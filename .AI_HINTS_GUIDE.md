# AI Hints Guide for UniFi Camera Proxy

This guide provides context and patterns to help AI assistants work effectively with this codebase.

## Project Overview

**Purpose**: Enables non-Ubiquiti cameras (especially Frigate) to work within the UniFi Protect ecosystem via WebSocket protocol emulation.

**Key Components**:
- **`unifi/main.py`**: Entry point, CLI argument parsing, camera factory
- **`unifi/cams/base.py`**: Core camera base class with protocol handling, snapshot management, event lifecycle
- **`unifi/cams/frigate.py`**: Frigate-specific integration via MQTT
- **`unifi/cams/managers/`**: Event lifecycle managers (SmartDetectEventManager, SmartMotionEventManager)
- **`unifi/cams/handlers/`**: Protocol handlers for snapshots, video streams, etc.
- **`unifi/cams/frigate_lib/`**: Frigate-specific utilities (descriptors, zones, snapshots, motion)

## Architecture Patterns

### 1. Mixin Composition Pattern

```python
class UnifiCamBase(ProtocolHandlers, VideoStreamHandlers, SnapshotHandlers, metaclass=ABCMeta):
    """Composition root for UniFi Protect camera emulation."""
```

**Pattern**: Multiple mixin classes provide orthogonal functionality. The base class doesn't call `super().__init__()` through MRO, so state is set directly.

**AI Hint**: When modifying behavior, add methods to the appropriate mixin rather than changing the base class.

### 2. Event Lifecycle Delegation

```python
# Base class owns lifecycle managers
self._smart_events = SmartDetectEventManager(...)
self._smart_motion = SmartMotionEventManager(...)

# Subclasses call trigger_* methods
await self.trigger_smart_detect_start(object_type=SmartDetectObjectType.PERSON)
```

**Pattern**: Event lifecycle is delegated to managers; base class provides thin delegation layer for backward compatibility.

**AI Hint**: Don't change the public `trigger_*` API signatures in base classâ€”add new functionality internally.

### 3. Frigate Integration Layers

```
FrigateCam (MQTT subscriber)
  â†“
FrigateEventHandlerMixin (protocol messages)
  â†“
RTSPCam (video streaming)
  â†“
UnifiCamBase (core protocol)
```

**Pattern**: Multiple inheritance with clear separation of concerns:
- **MQTT layer**: Receives Frigate events
- **Protocol layer**: Translates to UniFi messages
- **Streaming layer**: Handles RTSP/video

**AI Hint**: When adding Frigate features, add to `FrigateCam` or `FrigateEventHandlerMixin`, not base classes.

## Key Data Structures

### EventSmartDetect Message Structure

```python
{
    "from": "ubnt_avclient",
    "functionName": "EventSmartDetect",
    "inResponseTo": 0,
    "messageId": <monotonic counter>,
    "payload": {
        "clockMonotonic": <ms since boot>,
        "clockStream": <stream-relative ms>,
        "clockStreamRate": 1000,
        "clockWall": <epoch ms>,
        "eventId": <shared counter>,
        "edgeType": "enter|moving|leave|none",
        "objectTypes": ["person"],
        "displayTimeoutMSec": 300,
        "descriptors": [...],
        "zonesStatus": {"1": {"level": 75, "status": "enter"}},
        "smartDetectSnapshotFullFoV": "<path>",
        "smartDetectSnapshots": [...],
        "trackerIDAttrMap": {"700000": {"objectType": "person", "zone": [1]}}
    },
    "responseExpected": false,
    "timeStamp": "<ISO timestamp>",
    "to": "UniFiVideo"
}
```

**AI Hint**: `messageId` is shared across ALL message types per connection. `eventId` is per-camera but shared across all trackers.

### Descriptor Object

```python
{
    "trackerID": 700000,
    "objectType": "person",
    "coord": [x, y, w, h],  # in 640x360 sub-stream space
    "boxColor": "red|white",
    "confidenceLevel": 85,
    "firstShownTimeMs": <epoch>,
    "idleSinceTimeMs": 0,  # = firstShownTimeMs if stationary
    "stationary": false,
    "zones": [1, 2],  # most-recent-first
    "attributes": null,
    "lines": null,
    "loiterZones": null,
    "name": "",  # populated if face recognition matched
    "tag": ""
}
```

**AI Hint**: `coord` uses low-res sub-stream coordinates, not full-frame. Scale accordingly.

## Common Patterns for AI Tasks

### Adding a New Object Type

1. Add to `SmartDetectObjectType` enum in `smart_detect_manager.py`
2. Add to Frigate label mapping in `frigate_lib/descriptors.py`
3. Add to feature flags in `frigate.py`
4. Update any zone/status handling if needed

### Adding a New Zone

1. Update `zone_map` config in `FrigateCam.__init__`
2. Ensure `ZoneStatusTracker` includes new zone ID
3. Update `zonesStatus` to include all zones on every message

### Handling Doorbell Events

```python
# Check for doorbell in hello message
if "doorbell" in features and features["doorbell"]:
    # Route to doorbell handlers
    await self.send(self.gen_response("MCUEventMessage", payload={"eventType": "ring"}))
```

**AI Hint**: Doorbell uses `MCUEventMessage` function name, not `EventSmartDetect`.

### Snapshot Fetching

```python
# Prefer cached event-specific snapshots
crop, fov, heatmap = await self.fetch_snapshots_for_event(event_id)
# Falls back to live camera snapshot if not cached
```

**AI Hint**: Snapshot paths are cached per-tracker in `smart_event["tracker_snapshots"]`.

## Protocol-Specific Notes

### Clock Timing

- `clockMonotonic`: Elapsed ms since emulator start (consistent across all messages)
- `clockStream`: Fixed offset from `clockMonotonic` (~20-21s behind)
- `clockWall`: Epoch ms from Frigate's `frame_time * 1000`

**AI Hint**: Maintain a single `clockMonotonic` counter; don't reset per event.

### Zone Status Completeness

Every message must include ALL configured zones, not just active ones:

```python
zonesStatus = {}
for zone_id in configured_zone_ids:
    zonesStatus[str(zone_id)] = {
        "level": zone_level,  # 0-100
        "status": "enter|moving|leave|none"
    }
```

**AI Hint**: Don't omit zonesâ€”Protect expects all configured zones on every message.

### Stationary Object Handling

```python
if after.position_changes == 0:
    # Stationary object
    edgeType = "none"
    objectTypes = []  # Clear for stationary
    boxColor = "white"
```

**AI Hint**: Stationary objects have `edgeType: "none"` and don't populate `objectTypes`.

```
Error: Connection to MQTT server failed
```

**Fix**: Verify `--mqtt-host`, `--mqtt-port`, credentials.

```

```

**Fix**: Ensure Frigate HTTP API is accessible, or use MQTT snapshot events.

## AI-Specific Tips

1. **Don't modify base class `__init__`**: Use dependency injection or add optional parameters
2. **Preserve API signatures**: The `trigger_*` methods are stable; add complexity internally
3. **Use existing enums**: `SmartDetectObjectType` is the source of truth for object types
4. **Zone IDs are static**: Once configured in Protect, they don't changeâ€”use the `zone_map` config
5. **Message IDs are global**: One counter for all message types per connection
6. **Event IDs are per-camera**: Increment independently for each camera session
7. **Snapshots are cached**: Prefer cached event-specific snapshots over live camera snapshots
8. **Clock consistency**: Maintain monotonic clocks across all messages
9. **All zones on all messages**: Never omit zones from `zonesStatus`
10. **Stationary vs. active**: Distinguish via `edgeType` and `objectTypes` fields
